# 🚀 云打包快速开始（5分钟上手）

## 最简单的方法：GitHub Actions（推荐）

### 第一步：准备GitHub仓库

1. 在GitHub上创建一个新仓库
2. 将项目代码推送到仓库：

```bash
cd Kalman
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/你的仓库名.git
git branch -M main
git push -u origin main
```

### 第二步：启用GitHub Actions

1. 项目已经包含了 `.github/workflows/build-apk.yml` 文件
2. 推送代码后，GitHub会自动检测到工作流文件
3. 进入仓库的 "Actions" 标签页
4. 点击 "Build Android APK" 工作流
5. 点击 "Run workflow" 手动触发构建

### 第三步：等待构建完成

- ⏱️ 首次构建：约30-40分钟（需要下载Android SDK）
- ⏱️ 后续构建：约15-20分钟（使用缓存）

### 第四步：下载APK

1. 构建完成后，在Actions页面找到 "Artifacts"
2. 点击 "app-debug-apk" 下载
3. 解压后得到APK文件
4. 传输到手机并安装

## 其他快速方案

### 方案A：使用Docker（本地）

```bash
# 1. 构建Docker镜像
docker build -t kalman-builder .

# 2. 运行构建（APK会生成在 Kalman/bin/ 目录）
docker run --rm -v "$(pwd):/app" kalman-builder

# 3. 在 Kalman/bin/ 目录找到APK文件
```

### 方案B：使用GitLab CI/CD

1. 将代码推送到GitLab仓库
2. 项目已包含 `.gitlab-ci.yml` 文件
3. GitLab会自动开始构建
4. 在CI/CD页面下载APK

---

## ⚠️ 重要提示

1. **模型文件**：确保 `yolov8n.pt` 文件在项目根目录
2. **文件大小**：首次构建会下载大量依赖（约2-3GB）
3. **构建时间**：请耐心等待，首次构建较慢
4. **网络要求**：需要稳定的网络连接下载依赖

---

## 🐛 遇到问题？

### 构建失败？

1. 检查 `buildozer.spec` 配置
2. 查看构建日志中的错误信息
3. 确保所有必需文件都已提交到仓库

### APK无法安装？

1. 确保手机允许安装未知来源的应用
2. 检查APK是否针对正确的架构（arm64-v8a 或 armeabi-v7a）
3. 尝试使用 `adb install` 命令安装
